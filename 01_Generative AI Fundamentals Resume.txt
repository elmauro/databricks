		Vector Search 
			Es una base de datos vectorial que permite Busqueda semántica con embeddings vectoriales
			
			Se conecta con Mosaic AI Model Serving para embeddings. ignifica que Vector Search no genera los embeddings por sí mismo,
			sino que llama (o se conecta) al servicio de modelos (Model Serving) que los genera automáticamente.
			
			Sincronización automática con Delta Tables. Significa que Mosaic AI Vector Search puede mantener su índice vectorial actualizado automáticamente
			
			Busqueda con filtros de metadata. Es una búsqueda semántica (por similitud de embeddings) combinada con condiciones exactas sobre los metadatos.

	Riesgos Legales
	
		Privacidad de Datos
			El modelo puede exponer datos sensibles
			
			Solución:

			Usar Unity Catalog para control de acceso
			Anonimizar datos de entrenamiento
			Auditar outputs
			
		Propiedad Intelectual
			¿Quién es dueño del contenido generado?
			
			Solución:

			Revisar licencias de datos de entrenamiento
			Usar modelos con licencias claras
			Políticas claras de uso
			
	Riesgos de Seguridad
	
		Prompt Injection
			Usuarios maliciosos manipulan el LLM
			
			Solución:

			Implementar guardrails
			Validar inputs
			Usar Llama Guard
			
			
		Datos Sensibles en Entrenamiento
			Entrenar con datos confidenciales
		
			Solución:

			Limpiar datos antes de entrenar
			Usar técnicas de privacidad (DP, federated learning)
	
	
	Riesgos Éticos
		
		Sesgo (Bias)
			El modelo aprende sesgos de los datos
			
			Solución:

			Auditar datos de entrenamiento
			Medir fairness metrics
			Balance de datasets
			
		Alucinaciones (Hallucinations)
			El LLM inventa información falsa
			
			Tipos:
				Intrínseca: Contradice su propia respuesta
				Extrínseca: Inventa datos que no existen
				
			Solución:

				Usar RAG con datos verificados
				Implementar fact-checking
				Pedir citas/fuentes
				
				
	Riesgos Sociales y Ambientales
	
		Impacto Laboral
			Automatización puede eliminar empleos
			
			Solución:

				Re-entrenar empleados
				Usar IA como asistente, no reemplazo
				
		Consumo Energético
			Entrenar LLMs consume mucha energía
			
			Solución:

				Usar modelos pre-entrenados
				Fine-tuning eficiente
				Optimizar inferencia
